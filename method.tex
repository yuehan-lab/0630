\section{PhysCausal System Design}
\label{sec:method}

\subsection{System Pipeline}
\label{sec:pipeline}

PhysCausal operates as a five-step pipeline across client and server.%

\noindent\textbf{Step 1: Session initialization.}
The detection server issues a monitoring request specifying mode (passive or active) and
window length $T$ (default 30\,s).%
In active mode, the server pre-selects $t^* \in [\frac{T}{4}, \frac{3T}{4}]$ but
withholds it until the client confirms recording has started.%

\noindent\textbf{Step 2: Signal acquisition.}
The PhysCausal agent registers two listeners within the host app process:
(i)~\texttt{TYPE\_LINEAR\_ACCELERATION} via \texttt{SensorManager} at
\texttt{SENSOR\_DELAY\_FASTEST} ($f_s \approx 200$\,Hz on current hardware), and
(ii)~\texttt{View.OnTouchListener}, recording every \texttt{ACTION\_DOWN} timestamp
from the Android input system.%
In active mode, after the client acknowledges recording start, the server transmits $t^*$
over the encrypted channel; the app presents the challenge UI element at $t^*$ and
records the resulting touch timestamp.%

\noindent\textbf{Step 3: Preprocessing.}
Three server-side steps produce the model inputs.%
A moving-median filter (window 50\,ms) estimates background motion $\bar{\mathbf{a}}(t)$,
yielding the deviation signal
$\delta_{\mathrm{acc}}(t) = \|\mathbf{a}(t) - \bar{\mathbf{a}}(t)\|_2$
that preserves impulse transients while suppressing postural drift.%
A 4th-order Butterworth bandpass filter (1--50\,Hz) removes DC offset and aliased noise.%
Per-session z-score normalization removes device-to-device gain differences.%
The touch-event list is converted to a sparse indicator vector $\tau(t) \in \{0,1\}^T$
at the accelerometer sampling grid.%
Inter-touch intervals $\mathcal{I} = \{\Delta t_i = \tau_{i+1} - \tau_i\}$ are retained
as a third input stream.%

\noindent\textbf{Steps 4--5: PCCN inference and verdict.}
The Physical Causality Consistency Network (Section~\ref{sec:pccn}) classifies the session.%
The server returns a ternary label
(\textit{human} / \textit{autonomous agent} / \textit{overlay agent}) for downstream
enforcement.%

\subsection{Physical Causality Consistency Network (PCCN)}
\label{sec:pccn}

PCCN is purpose-built around the physics of finger-touch causality.%
Rather than applying generic cross-modal attention, each architectural component encodes
a specific physical prior about the accelerometer impulse response.%
Fig.~\ref{fig:pccn} shows the full architecture.%

\noindent\textbf{Stream 1: Accelerometer Encoder.}
The deviation signal $\delta_{\mathrm{acc}}(t) \in \mathbb{R}^T$
($T{=}6{,}000$ samples for 30\,s at 200\,Hz) is divided into
non-overlapping patches of $p{=}40$ samples (200\,ms each), yielding $N_p{=}150$ tokens.%
Each patch is projected to $d{=}128$ dimensions; learnable positional embeddings are added.%
An $L{=}6$ layer Transformer encoder ($H{=}4$ heads, feed-forward width 512) produces
$E_{\mathrm{acc}} \in \mathbb{R}^{150 \times 128}$.%

\noindent\textbf{Stream 2: Touch-Event Encoder.}
Each of the $N_e$ touch events $\{t_i^e\}$ is encoded as a learned event embedding
$\mathbf{e}_{\mathrm{evt}} \in \mathbb{R}^{128}$ plus a continuous sinusoidal positional
encoding derived from its absolute timestamp within the 30-s window, yielding
$E_{\mathrm{touch}} \in \mathbb{R}^{N_e \times 128}$.%

\subsubsection{Physics-Constrained Causal Window Attention (PC-CWA)}
\label{sec:pccwa}

Generic cross-modal attention---as in Crossformer~\cite{crossformer2023}, which allows
queries to attend across all temporal positions---allows each touch-event query to attend
to any part of the accelerometer stream, including regions temporally unrelated to the touch.%
This is physically wrong: the causal impulse response of finger contact is localized to
$[0, T_{\mathrm{win}}]$ after the touch and decays monotonically.%
Unlike CausalFormer~\cite{causalformer2024}, which \textit{discovers} causal structure
post-hoc via learned attention, PC-CWA \textit{encodes} known physical causal structure
as a hard architectural constraint, providing stronger inductive bias and higher
sample efficiency.%
PC-CWA encodes these two physical priors directly into the attention mechanism.%

\noindent\textbf{Causal mask.}
Let patch $j$ span the time interval $[s_j, s_j + p/f_s]$.%
For touch event $i$ at time $t_i^e$, we define the binary causal mask:
\begin{equation}
M_{ij} =
\begin{cases}
1 & \text{if } s_j \in [t_i^e,\; t_i^e + T_{\mathrm{win}}] \\
0 & \text{otherwise}
\end{cases}
\label{eq:mask}
\end{equation}
where $T_{\mathrm{win}} = 80$\,ms (4 patches at 200\,Hz with $p{=}40$).%
Masked positions receive $-\infty$ before softmax, preventing any attention from being
allocated outside the physically valid window.%

\noindent\textbf{Exponential decay prior.}
Within the causal window, physical impulse responses decay exponentially~\cite{biomotouch2026}.%
We weight each attention score by a learnable decay:
\begin{equation}
w_{ij} = \exp\!\left(-\frac{s_j - t_i^e}{\tau_0}\right), \quad
\tau_0 \in \mathbb{R}^+\;\text{(learnable, initialized to 40\,ms)}
\label{eq:decay}
\end{equation}
The full PC-CWA attention is:
\begin{equation}
A_{ij} = \mathrm{Softmax}\!\left(
  \frac{Q_i K_j^\top}{\sqrt{d}} \cdot \mathbf{1}[M_{ij}=1] + (-\infty)\cdot\mathbf{1}[M_{ij}=0]
\right) \cdot w_{ij}
\label{eq:pccwa}
\end{equation}
\begin{equation}
E_{\mathrm{causal}} = A\, V_{\mathrm{acc}} \in \mathbb{R}^{N_e \times d}
\label{eq:ecausal}
\end{equation}
where $Q = E_{\mathrm{touch}} W^Q$, $K = E_{\mathrm{acc}} W^K$,
$V = E_{\mathrm{acc}} W^V$ are learned projections.%
The learned $\tau_0$ converges to the physical impulse decay constant, providing
a data-driven validation of the physical model.%

\subsubsection{Physical Causal Score (PCS)}
\label{sec:pcs}

The attention weights $\{A_{ij}\}$ encode the model's belief about causal coupling.%
We aggregate them into a differentiable, per-event physical causal score:
\begin{equation}
\mathrm{PCS}_i = \sum_{j:\,M_{ij}=1} A_{ij} \;\in\; [0,1]
\label{eq:pcs}
\end{equation}
$\mathrm{PCS}_i$ is the total attention mass allocated within the causal window for
touch event $i$.%
For human sessions, $\mathrm{PCS}_i$ is high at all events (impulse present).%
For autonomous agent sessions, $\mathrm{PCS}_i$ is uniformly low (no impulse anywhere).%
For overlay agent sessions, $\mathrm{PCS}_i$ is high at genuine human touches but
low at the server-challenged position $t^*$ (agent injection, no impulse).%
The session-level feature $\bar{\mathrm{PCS}} = N_e^{-1}\sum_i \mathrm{PCS}_i$ and
the per-event PCS vector are both passed to the classification head.%
Unlike post-hoc attention visualization, PCS is part of the forward pass---it is an
interpretable, backpropagable feature.%

\subsubsection{Impulse Spectral Prototype Module (ISPM)}
\label{sec:ispm}

Not all impulse-like signals in the causal window originate from real finger contact.%
An A1 adversary attaches a vibration motor to the chassis, producing a narrowband
sinusoidal oscillation in the causal window that the causal mask cannot reject.%
ISPM distinguishes impulse types by their spectral fingerprint.%

For each touch event $i$, we extract the magnitude spectrum of $\delta_{\mathrm{acc}}$
within the causal window:
\begin{equation}
\mathbf{F}_i = |\mathrm{FFT}(\delta_{\mathrm{acc}}[t_i^e : t_i^e + T_{\mathrm{win}}])| \in \mathbb{R}^{B}
\label{eq:fft}
\end{equation}
with $B{=}8$ log-spaced frequency bins spanning 1--50\,Hz (each bin covers a
physically meaningful band: 1--4, 4--8, 8--15, 15--25, 25--35, 35--42, 42--48, 48--50\,Hz).%

A learnable prototype bank $\mathbf{P} \in \mathbb{R}^{K \times B}$ ($K{=}3$) holds
three spectral templates:%
\begin{itemize}[leftmargin=*,topsep=2pt,itemsep=0pt]
  \item $\mathbf{p}_1$: broadband (human finger impact---energy spread across 1--50\,Hz)
  \item $\mathbf{p}_2$: narrowband (vibration motor---single-bin energy concentration)
  \item $\mathbf{p}_3$: flat noise (agent injection---no energy above noise floor)
\end{itemize}
Soft prototype assignment uses squared Euclidean distance with a temperature $T_p$:
\begin{equation}
\mathbf{s}_i = \mathrm{Softmax}\!\left(-\frac{\|\mathbf{F}_i - \mathbf{P}\|_2^2}{T_p}\right) \in \mathbb{R}^K
\label{eq:proto_assign}
\end{equation}
The prototype feature for event $i$ is $\mathbf{e}_{\mathrm{proto},i} = \mathbf{s}_i^\top \mathbf{P} \in \mathbb{R}^B$.%
The session-level prototype representation
$E_{\mathrm{proto}} = \mathrm{GAP}(\{\mathbf{e}_{\mathrm{proto},i}\}_{i=1}^{N_e}) \in \mathbb{R}^B$
is projected to $\mathbb{R}^d$ via a learned linear layer and concatenated into the
classification head.%
After training, $\mathbf{P}$ is inspectable: $\mathbf{p}_1$ should show broadband
energy concentration; $\mathbf{p}_2$ a sharp peak at the motor driving frequency;
$\mathbf{p}_3$ a flat profile at noise floor---a physics-grounded sanity check
that requires no annotation.%

\noindent\textbf{Stream 3: Timing Distribution Encoder.}
Inter-touch intervals $\mathcal{I}$ are embedded via two parallel branches:
(i)~a log-normal head computing $[\mu_{\log}, \sigma_{\log}, \mathrm{skew}(\log \mathcal{I})]$
and (ii)~a bimodality head computing the bimodality coefficient $B$~\cite{bimodality1985}
and the sub-300\,ms event ratio (characteristic of LLM cloud inference latency).%
Both outputs are concatenated and projected to $E_{\mathrm{timing}} \in \mathbb{R}^{128}$.%

\noindent\textbf{Passive head.}
$[\mathrm{GAP}(E_{\mathrm{acc}}) \| E_{\mathrm{timing}}] \in \mathbb{R}^{256}$
is passed through a two-layer MLP (hidden 128, dropout 0.2) to a 2-class logit
(human vs.\ any agent).%
Passive mode does not use PC-CWA or ISPM (no touch-event queries needed for
background accelerometer analysis).%

\noindent\textbf{Active head.}
All streams are fused:
\begin{equation}
\mathbf{h} = \bigl[
  \mathrm{GAP}(E_{\mathrm{acc}}) \;\|\;
  \mathrm{GAP}(E_{\mathrm{causal}}) \;\|\;
  E_{\mathrm{proto}} \;\|\;
  \bar{\mathrm{PCS}} \;\|\;
  E_{\mathrm{timing}}
\bigr] \in \mathbb{R}^{4d+1}
\label{eq:fusion}
\end{equation}
A two-layer MLP (hidden 256, dropout 0.2) maps $\mathbf{h}$ to 3-class logits.%
The explicit $\bar{\mathrm{PCS}}$ term allows the head to directly read off the
session-level causal signal, while event-level PCS information is propagated
via $\mathrm{GAP}(E_{\mathrm{causal}})$.%

\subsection{Three-Stage Training}
\label{sec:training}

\noindent\textbf{Stage 1: Masked accelerometer pre-training.}
The Transformer encoder is pre-trained on a large corpus of unlabeled smartphone
accelerometer recordings using masked autoencoding (MAE)~\cite{he2022mae,patchtst2023,simmtm2023}.%
A random 75\% of patch tokens are masked; the encoder processes visible tokens, and a
lightweight decoder reconstructs masked patches.%
Objective:
\begin{equation}
\mathcal{L}_{\mathrm{MAE}} = \frac{1}{|\mathcal{M}|} \sum_{i \in \mathcal{M}}
  \|\hat{\mathbf{p}}_i - \mathbf{p}_i\|_2^2.
\label{eq:mae}
\end{equation}
This stage teaches the encoder temporal structure of device motion without label supervision.%

\noindent\textbf{Stage 2: Supervised contrastive fine-tuning.}
We fine-tune the encoder with three complementary objectives:
\begin{equation}
\mathcal{L}_{\mathrm{S2}} = \mathcal{L}_{\mathrm{CE}}
  + \lambda_1 \mathcal{L}_{\mathrm{TFC}}
  + \lambda_2 \mathcal{L}_{\mathrm{SCL}}
  + \lambda_3 \mathcal{L}_{\mathrm{proto}}
\label{eq:stage2}
\end{equation}
$\mathcal{L}_{\mathrm{CE}}$ is cross-entropy on class labels.%
$\mathcal{L}_{\mathrm{TFC}}$ is time-frequency consistency contrastive loss~\cite{tfc2022},
enforcing agreement between time-domain and frequency-domain views of the same session.%
$\mathcal{L}_{\mathrm{SCL}}$ is supervised contrastive loss~\cite{supcon2020}, pulling
same-class session embeddings together and pushing different-class embeddings apart.%
$\mathcal{L}_{\mathrm{proto}} = -\frac{1}{N_e}\sum_i \log s_{i,c_i}$ is prototype
assignment cross-entropy, where $c_i \in \{1,2,3\}$ is the ground-truth impulse type
(human, motor, or noise) for event $i$, supervising ISPM directly.%
Default weights: $\lambda_1 = 0.5$, $\lambda_2 = 0.1$, $\lambda_3 = 0.2$.%

\noindent\textbf{Stage 3: Cross-modal head training with Overlay-Aware Loss.}
With the Transformer encoder frozen, we train PC-CWA, ISPM, the PCS layer, and both
classification heads on labeled touch--accelerometer pairs.%
For overlay agent sessions, standard cross-entropy loss treats the entire session as
a single label, which does not penalize the model for misattributing PCS at $t^*$.%
We introduce the \textit{Overlay-Aware Loss} (OAL):
\begin{equation}
\mathcal{L}_{\mathrm{OAL}} = \frac{1}{|\mathcal{S}_{\mathrm{ov}}|}
  \sum_{s \in \mathcal{S}_{\mathrm{ov}}}
  \max\!\bigl(0,\; \mathrm{PCS}_{t^*}^{(s)} - \gamma\bigr)
\label{eq:oal}
\end{equation}
where $\mathcal{S}_{\mathrm{ov}}$ is the set of overlay agent sessions and
$\gamma = 0.3$ is a margin.%
OAL penalizes the model whenever it assigns high PCS to the challenged event $t^*$ in
an overlay session, forcing it to localize the injection-specific causality failure
rather than averaging over all events.%
Stage 3 objective:
\begin{equation}
\mathcal{L}_{\mathrm{S3}} = \mathcal{L}_{\mathrm{CE}} + \lambda_4 \mathcal{L}_{\mathrm{OAL}},
\quad \lambda_4 = 0.3.
\label{eq:stage3}
\end{equation}
We use AdamW~\cite{adamw2019} with a cosine learning rate schedule (peak $10^{-4}$,
warmup 5 epochs) and label smoothing 0.1 throughout.%

\subsection{Session Aggregation and Gated Mode Fusion}
\label{sec:aggregation}

For windows exceeding 30\,s, the session is split into non-overlapping 30-s segments;
the session-level verdict is the majority vote of segment predictions (ties broken by
maximum confidence class).%
A scalar gate $g = \sigma(W_g \mathbf{h})$ controls the active-mode contribution at
inference: for a passive-mode session (no $t^*$), the gate zeroes out the
$E_{\mathrm{causal}}$ and $\bar{\mathrm{PCS}}$ terms, reducing the active head to a
timing-aware passive classifier without requiring a separate model.%

\subsection{Complexity and Deployment}
\label{sec:complexity}

PCCN has 3.8\,M parameters.%
Server-side inference on a 30-s window takes 47\,ms on a single Intel Xeon core,
adding negligible latency to any authentication flow.%
The client-side agent consumes 4.2\,mAh over a 30-s session
($\approx$1.8\% of idle power), runs zero dangerous permissions, and requires
no background service.%
